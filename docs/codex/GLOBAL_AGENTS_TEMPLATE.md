# Global Working Agreements

- Work in small, controlled slices.
- Inspect before editing.
- Preserve unrelated work.
- Never claim success before verification.
- Do not commit unless the task explicitly authorizes it.
- Do not deploy or modify production without explicit, scope-specific approval.
- Stop when scope, identity, environment, authorization, or command behavior is uncertain.
- Never expose credentials, tokens, customer data, private identifiers, or production exports.
- Prefer the smallest safe implementation over broad rewrites.
- Do not hide failures with retries, skips, broad timeout changes, or permissive fallbacks.
- Report branch, HEAD, files changed, tests, production actions, blockers, risks, and the recommended next action.
- Keep dynamic project state in the repository's current-state document rather than repeating it in every task prompt.
- Read repository and nearest directory `AGENTS.md` files before changing code.
- Follow the user's requested approval gates and stop conditions exactly.
