Yes, that could work as a simple standalone site.

The angle should not be just:

Random jokes

because there are already tons of those.

The stronger angle is:

Context-based jokes that are actually usable

Examples:

Workplace jokes
Dad jokes
Cleaning jokes
Retail jokes
Manager jokes
Customer service jokes
Gaming jokes
Card shop jokes
Teacher jokes
Icebreaker jokes
Birthday jokes
Holiday jokes

The useful feature would be:

I need a joke about:
[workplace stress]

Tone:
[clean / dad joke / sarcastic / wholesome / gamer]

Audience:
[coworkers / employees / customers / friends]

Generate

You could also have categories like:

Safe for Work
No Profanity
Short One-Liners
Meeting Openers
Team Morale
Customer Service Humor
Cleaning Humor
Retail Humor
Gaming Humor

The bigger opportunity is quality filtering:

Users vote
Best jokes rise
Bad jokes disappear
AI generates variants
Human approval improves library

So the site becomes:

Joke Search Engine
+
AI Joke Generator
+
Community Ranking

Possible monetization later:

Free basic jokes
Premium AI joke generator
Team/business accounts
Slack/Discord/Teams bot
API access

I’d put it in the idea folder as a micro-SaaS / content site, not a main project yet.

Name idea:

JokeForge AI

or:

Punchline Engine

The simple MVP would be:

Search jokes
Filter by category/tone
Generate new joke
Thumbs up/down
Save favorites

That’s enough to validate whether people actually use it.
