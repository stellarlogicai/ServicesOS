# SLAI Website Roadmap

## Purpose

The SLAI website should not be built until ServicesOS is stable enough for beta testing.

Priority:

```text
ServicesOS Web
    ↓
Employee App
    ↓
Wife Beta Testing
    ↓
Bug Fixes
    ↓
Customer Beta
    ↓
SLAI Website
    ↓
Public Launch
```

The website should showcase REAL features, screenshots, and workflows instead of future promises.

---

# Website Goals

The SLAI website should:

* Explain who SLAI is
* Showcase ServicesOS
* Allow customers to request demos
* Provide app download links
* Provide login access
* Host support and documentation
* Grow as more products are released
